M6: AI Overhaul — Architecture Overview

Three pillars, two AI tiers, one cohesive experience

Two-Tier AI Model

Tier 1: BYOK / On-Device

Who: Users who bring their own API key

Where AI runs:

  • 📱 On-device — FoundationModels (iOS 26+) for summaries, key points, scoring
  • 📱 Direct API calls — App calls Anthropic/OpenAI directly with user's key for chat, brief

Background enrichment: ✅ Yes — on new articles, app enriches locally

Cost to you: $0

Rate limits: None (their key, their bill)

Server role: Storage & sync only

Tier 2: Subscription (Server AI)

Who: Users who pay for NebularNews AI

Where AI runs:

  • ☁️ Server-side — Workers calls AI providers with platform keys
  • 📱 On-device fallback — FoundationModels when available

Background enrichment: ✅ Yes — server enriches on feed poll

Cost model:

  • Subscription tiers (daily/weekly token budgets)
  • Opt-in overages at API cost + 10%
  • Hard cap if user opts out of overages

Rate limits: Enforced per-user via ai_usage table

Three Pillars of Improvement

💬

Chat Experience

  • Streaming responses (SSE from Workers)
  • Conversation memory across articles
  • Richer context — related articles, user reading history
  • Follow-up suggestions after each response
  • On-device chat for BYOK (direct API calls)
🧠

Intelligence Layer

  • Topic clustering — group related articles
  • Trend detection — "AI regulation trending in your feeds"
  • Personalized scoring v2 — learns from reads, saves, reactions
  • Smart recommendations — "You might also like..."
  • Reading insights — weekly reading patterns
📰

Content Generation

  • Brief 2.0 — scheduled push, per-topic, configurable depth
  • Auto-enrichment — summarize on poll (background)
  • Feed digests — AI summary per feed/topic
  • Configurable summary style — wire up length/style settings
  • Key points on all articles (not just on-demand)

Infrastructure (supports everything above)

Metering & Billing
  • Token tracking per request (ai_usage table)
  • Daily/weekly budget enforcement
  • Overage calculation (cost + 10%)
  • Usage dashboard in Settings
  • Subscription tier management
Streaming Infrastructure
  • SSE endpoint on Workers
  • URLSession streaming on iOS
  • Incremental UI rendering
  • Error recovery mid-stream
On-Device AI Pipeline
  • Background processing on article fetch
  • FoundationModels integration (iOS 26+)
  • Direct Anthropic/OpenAI calls from app
  • Result sync to server
Retry & Reliability
  • Exponential backoff on failed enrichment
  • Job queue with attempt tracking
  • Fallback provider chain
  • Graceful degradation (no AI → still readable)
