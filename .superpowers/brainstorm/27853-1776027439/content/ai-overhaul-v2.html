M6: AI Overhaul — Revised Architecture

Four pillars, two AI tiers, MCP-connected

Two-Tier AI Model (revised)

Tier 1: BYOK

Who: Users who bring their own API key

Where AI runs:

  • ☁️ Server-proxied — App sends key in headers → Workers proxies to AI provider
  • 📱 FoundationModels only — On-device Apple AI when available (iOS 26+), for lightweight enrichment

Background enrichment: ❌ No background device processing

Batch enrichment: ✅ User-initiated "Enrich All" while app is open, routed through server

Cost to you: Workers compute only (~$0)

Rate limits: None (their key, their bill)

Tier 2: Subscription (Server AI)

Who: Users who pay via RevenueCat subscription

Where AI runs:

  • ☁️ Server-side — Workers calls AI providers with platform keys
  • 📱 FoundationModels fallback — On-device when available

Background enrichment: ✅ Server enriches on feed poll

Cost model:

  • RevenueCat subscription tiers (daily/weekly token budgets)
  • Opt-in overages at API cost + 10%
  • Hard cap if user opts out

Rate limits: Enforced per-user via ai_usage table

Four Pillars

💬

1. Chat Experience

  • Streaming responses — SSE from Workers, incremental rendering in iOS
  • Conversation memory — cross-article thread continuity
  • Richer context — related articles, user reading history in context window
  • Follow-up suggestions — generated after each response
  • Better prompts — expert analyst with markdown citations
🧠

2. Intelligence Layer

  • Topic clustering — group related articles automatically
  • Trend detection — "AI regulation trending in your feeds"
  • Scoring v2 — learns from reads, saves, dismisses, time spent
  • Smart recommendations — "You might also like…"
  • Reading insights — weekly patterns, topic breakdown
📰

3. Content Generation

  • Brief 2.0 — scheduled push, per-topic, configurable depth
  • Auto-enrichment — summarize on poll for subscribers
  • Batch enrichment — user-initiated "Enrich All" for BYOK
  • Configurable style — wire up summary length/style settings
  • Feed digests — AI summary per feed or topic
🔌

4. MCP Server

  • NebularNews as MCP server — connect to Claude Desktop, Cursor, etc.
  • Tools exposed: search articles, get summaries, read article, list feeds, get brief, ask about today's news
  • Resources: article content, feed lists, briefs, reading history
  • Auth: Bearer token from existing session
  • Transport: Streamable HTTP (remote MCP over HTTPS)

Infrastructure

Metering & Billing
  • Token tracking per request
  • Daily/weekly budget enforcement
  • Overage calc (cost + 10%)
  • Usage dashboard in Settings
  • RevenueCat subscription management
Streaming
  • SSE from Workers (chat)
  • URLSession streaming on iOS
  • Incremental markdown rendering
  • Error recovery mid-stream
Reliability
  • Retry with exponential backoff
  • Job queue with attempt tracking
  • Provider fallback chain
  • Graceful degradation

Suggested Build Order

Phase A — Foundation: Metering/rate-limiting, streaming infrastructure, RevenueCat integration, retry logic

Phase B — Chat & Content: Streaming chat, Brief 2.0, auto/batch enrichment, configurable style

Phase C — Intelligence: Topic clustering, trend detection, scoring v2, reading insights

Phase D — MCP: MCP server on Workers, tool definitions, auth, Claude Desktop integration
