M6: AI Overhaul — Architecture v3

Three AI tiers, four pillars, MCP-first

Three-Tier AI Model

Free: On-Device

Who: Any user (opt-in)

How: FoundationModels (iOS 26+)

Capabilities:

  • Summarization
  • Key points
  • Basic scoring
  • Tag suggestions

Limits: Device capability only

Network: None — fully offline

Cost to you: $0

BYOK: Bring Your Own Key

Who: Power users with AI API keys

How: App sends key in headers → Workers proxies to AI provider

Capabilities:

  • Everything on-device can do, plus:
  • Chat (streaming)
  • News briefs
  • Batch enrichment (user-initiated)
  • Advanced scoring

Limits: None (their key, their bill)

Cost to you: Workers compute (~$0)

Subscription: Server AI

Who: Users who pay via StoreKit IAP

How: Workers uses platform AI keys

Capabilities:

  • Everything BYOK can do, plus:
  • Auto-enrichment on feed poll
  • Scheduled briefs (push)
  • Priority processing

Limits: Daily/weekly token budgets per tier

Overages: Opt-in at API cost + 10%

Billing: StoreKit 2 (native IAP)

Four Pillars (build order)

PHASE A
⚙️

1. Infrastructure

Foundation for everything else

  • Streaming pipeline — SSE from Workers + URLSession streaming on iOS
  • Metering — token tracking per request, populate ai_usage table
  • Rate limiting — daily/weekly budget enforcement, overage tracking
  • Retry logic — exponential backoff, attempt tracking, provider fallback
  • StoreKit 2 IAP — subscription tiers, entitlement verification
  • On-device pipeline — FoundationModels integration, result sync to server
  • Tier resolution — on-device → BYOK → subscription → no AI (graceful)
PHASE B
🔌

2. MCP Server

The killer differentiator

  • Streamable HTTP transport — remote MCP over HTTPS on Workers
  • Tools:
    • search_articles — FTS5 search across all articles
    • get_article — full content + enrichment
    • get_brief — today's news brief
    • list_feeds — user's subscriptions
    • ask_about_news — multi-article chat context
    • get_trending_topics — what's hot in user's feeds
  • Resources: article content, feed OPML, reading history
  • Auth: Bearer token (existing session system)
PHASE C
💬

3. Chat & Content

Builds on streaming infra from Phase A

  • Streaming chat — words appear as generated (uses SSE pipeline)
  • Conversation memory — cross-article thread continuity
  • Follow-up suggestions — after each AI response
  • Brief 2.0 — scheduled push, per-topic, configurable depth
  • Auto-enrichment — summarize on poll (subscribers)
  • Batch enrichment — user-initiated "Enrich All" (BYOK)
  • Configurable style — wire up summary length/style settings
PHASE D
🧠

4. Intelligence

The "it learns you" layer

  • Topic clustering — group related articles automatically
  • Trend detection — "AI regulation trending in your feeds"
  • Scoring v2 — learns from reads, saves, dismisses, time spent
  • Smart recommendations — "You might also like…"
  • Reading insights — weekly patterns, topic breakdown
